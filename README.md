## About

I focus on document understanding systems that are useful beyond demos:

- OCR and layout analysis for scanned PDFs, reports, tables, charts, and mixed image-text documents
- PDF-to-Markdown conversion with structure, reading order, formulas, tables, and figures preserved
- benchmark tooling for evaluating document parsing and OCR systems under reproducible settings
- multimodal data engineering for building cleaner datasets, better evaluation sets, and stronger feedback loops

My current view: as model-architecture gains become harder to extract, the next round of model quality will depend heavily on data engineering: dataset construction, cleaning, labeling, synthetic data, evaluation design, and error-driven iteration.

## Current Focus

- Comparing OCR and document parsing systems such as PaddleOCR, MinerU, and GLM-OCR
- Studying papers and technical reports with NotebookLM, then turning notes into engineering experiments
- Building small tools for OCR result cleaning, layout recovery, table extraction, and error-case analysis
- Evaluating how document AI outputs can support search, RAG, and knowledge-base workflows

## What I Care About

- structured outputs that can be used directly in RAG and knowledge workflows
- reproducible local pipelines instead of one-off benchmark screenshots
- transparent evaluation: clear datasets, clear metrics, and inspectable failure cases
- data-centric iteration: error analysis first, optimization second
- engineering tradeoffs between quality, speed, cost, and controllability
